6. Reference pattern (copy from vault)

Every route follows this shape:

auth (requireAuth / requireClientUser)
  → validate (Zod safeParse)
  → service / domain logic in route
  → ask repo
  → ok() / fail()

Wrapped in withRoute(...) so any throw becomes a generic { error: "internal error" } 500 + a logged [route METHOD path] line.

Best example to copy: app/api/vault/items/route.ts (full CRUD on the kernel + repos + Zod).

8. Collision rules

  • One owner per repo file. Whoever creates xRepo.ts owns it. Others import — never recreate.
  • Schemas append-only. Add new schemas at the end of lib/validation/schemas.ts. Don't reorder existing ones.
  • Never edit a money value during refactor. Prices, splits, hard-stops are CLAUDE.md law. Relocate, don't change.
  • Branch off Amir-Dev after the checkout commit lands. Otherwise the new dev won't see the uncommitted repos.
  • Don't touch checkout files while the current owner is still on it.
  • Verify gate green (tsc + lint + test) after every step. Red gate = bug you're shipping.
