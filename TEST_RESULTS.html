The 5 failures — what & why

One shared root cause. All 5 failing tests call loginAs(), which performs the app's cross-host auth handoff to clientUrl("/auth/handoff"). In .env.test, NEXT_PUBLIC_CLIENT_HOST=app.localhost:3000 — but the Playwright dev server runs on port 3100, and app.localhost is not in /etc/hosts. So after login the browser is redirected to an unreachable host, lands on a Next.js error page ("missing required error components, refreshing…"), and the test times out waiting for the URL. This is a test-infra mismatch, not a code defect.

The 43 skipped

Tests tagged @pro / @sales / @admin run in the other three Playwright projects (the consumer app project excludes them via grepInvert). They need the partner / sales / admin subdomains, which require /etc/hosts entries that aren't set on this machine.

Make the whole suite green (one-time infra setup)

Not a code change — the e2e login + portal specs need the multi-host setup the app's auth handoff expects:

# 1. add the subdomains to /etc/hosts (matches the host the handoff targets)
echo "127.0.0.1 app.localhost pro.localhost sales.localhost admin.localhost" \
  | sudo tee -a /etc/hosts

# 2. align the dev-server port with NEXT_PUBLIC_CLIENT_HOST in .env.test
#    (either run the test server on :3000, or set the *_HOST vars to :3100)

# 3. seed staging (already done for this run)
TEST_ALLOW_SHARED_DB=1 npm run test:db:seed

# 4. run every project (app + pro + sales + admin)
npx playwright test

With that in place the login-handoff and the @pro/@sales/@admin specs can pass too. None of it changes application code.
